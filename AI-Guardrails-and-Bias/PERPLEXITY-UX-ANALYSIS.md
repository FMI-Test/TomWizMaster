## 🏆 The Trinity UX Pattern (THREE = Power of Three)

### The Implicit Trinity Reference

Perplexity's feedback UI is a masterclass in **choice architecture**:

```
┌─────────────────────────────────────────────────────┐
│  Response Content...                                │
│                                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
                                [ 👍 ]  [ 👎 ]  [ ⋯ ]
                                                  │
                                                  └──┬── 💾 Save
                                                     ├── 🚩 Report  
                                                     └── 🗑️ Delete
```

### Why This Design is Perfect

| Principle | Implementation |
|-----------|----------------|
| **Non-Intrusive** | Bottom-left, doesn't block content |
| **Not Forcing** | User chooses IF to engage, not coerced |
| **Respectful Framing** | "Help us know how we're doing" (gratitude, not demand) |
| **THREE Core Actions** | Binary (👍/👎) + Extended (⋯) |
| **THREE Extended Options** | Save / Report / Delete |
| **Founder Character** | Elegant minimalism, like Foundation's Hari Seldon 🤪 |

### The Trinity → THREE Connection

> "I love Trinity... 3 options, the third a three dots dropdown with THREE options"

| Layer | Count | Options |
|-------|-------|---------|
| **Primary** | 3 | 👍 / 👎 / ⋯ |
| **Secondary** | 3 | Save / Report / Delete |
| **Total Paths** | 5 | Two immediate + three extended |

**Design Philosophy**: Give user **power** without **overwhelm**. The three-dot pattern is universal (iOS, Android, Windows) — learned behavior, zero friction.
