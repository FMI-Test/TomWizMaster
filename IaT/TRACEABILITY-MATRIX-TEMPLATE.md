# Traceability Matrix — Requirements ↔ Components ↔ Tests

## Overview
Map SR requirements to platform/app components and verification artifacts to ensure end‑to‑end coverage.

## Template
| Requirement ID | Requirement | Layer (System/Platform/App) | Component(s) | Test Case(s) | Risk(s) | Mitigation(s) |
|----------------|------------|------------------------------|--------------|--------------|---------|----------------|
| SR‑001 | <desc> | Platform | <comp> | <tests> | <risks> | <plan> |
| SR‑002 | <desc> | App | <comp> | <tests> | <risks> | <plan> |

## Usage
- Fill for major SR items; ensure each has tests and mitigations.
- Update when X → Y substitutions occur; re‑map components and tests.
- Link to audit entries when high‑stake changes are made.
