# Policy Cost Change Analysis

## Component

- Current state -> Structured policy-cost packet for the 1979 Iran counterfactual and downstream-war analysis.

## Layer

- App

## Context

- The repository already contains fragments linking Iran 1979, Afghanistan, Syria, and October 7, but not a single structured document that converts those themes into explicit cost classes and bounded counterfactuals.

## Interfaces And Contracts

- Inputs/Outputs:
  - Inputs: public-source ranges, repository themes, working assumptions, and confidence labels.
  - Outputs: main markdown brief, requirements document, options document, and traceability matrix.
- Guardrails:
  - No single-cause determinism presented as settled fact.
  - No dehumanizing or inciting rhetoric.
  - No unsupported point estimates where only rough bands are defensible.
- SR Constraints:
  - Transparency, neutrality, bounded counterfactuals, and auditability.

## Pros / Cons

- Pros:
  - Consolidates scattered policy-cost themes into one canonical document.
  - Makes large claims easier to review because assumptions are explicit.
  - Adds labor-market opportunity cost, which improves practical readability.
- Cons:
  - Some quantitative ranges remain coarse until more primary-source work is added.
  - Counterfactual sections will always carry interpretation risk.

## Cost / Time / Value Alignment

- Cost: Low documentation cost now; higher research cost if the file is later expanded into a fully sourced paper.
- Timeline: Immediate for structure; moderate for any future source-deepening.
- Value: High, because the file transforms a broad grievance into an analyzable framework.

## Risks And Mitigations

- Security/Compliance: The subject is politically charged and can drift into propaganda.
- Operational: Coarse numbers may be read as exact totals if confidence is not emphasized.
- Mitigations:
  - Use explicit confidence tags.
  - Use ranges instead of artificial precision.
  - Keep normative conclusions separated from the evidence band table.

## Option Mapping

- A (Recommended): Structured brief with explicit evidence bands and counterfactual limits.
- B (Simple): Short rhetorical note without structured support.
- C (Flexible): Reusable template for future policy-cost chains.

## Rollout And Rollback

- Test Plan:
  - Verify cross-links.
  - Check markdown validity.
  - Confirm each major claim family maps to a requirement.
- Rollout Plan:
  - Publish the main brief first.
  - Add deeper sourcing later if needed.
- Rollback Plan:
  - Remove the packet and keep only the earlier fragmented references if the topic is rejected.
