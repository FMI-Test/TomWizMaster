# Emoji Compression — Quick Status Language

**Version:** 1.0
**Status:** Live
**Last Updated:** 2026-01-25

Purpose: Provide a concise, domain‑agnostic emoji palette to compress status and intent — helpful for Gen Z/Alpha workflows — while keeping audit integrity, neutrality, and accessibility.

## Principles
- Clarity first: one emoji line → structured bullets follow.
- Accessibility: include text labels; don’t rely on color/emoji alone.
- Neutrality: avoid culturally loaded symbols for decisions.
- Audit: use AAK signals; bind reasoning summaries when active.

## Palette (Recommended)
- **✅ OK:** passed, safe, reversible
- **❌ Blocked:** failed, cannot proceed
- **⚠️ Caution:** risky; needs review
- **🛑 Stop:** hard stop (Red action)
- **⏳ Pending:** waiting, in progress
- **🧠 Context:** reasoning or background
- **💡 Insight:** idea, hypothesis
- **🔧 Fix:** actionable step/change
- **🧪 Test:** verification needed
- **⚡ Quick:** minimal path
- **🔒 Secure:** compliance/security note
- **🧭 Direction:** decision or choice
- **📎 Link:** reference attached
- **📌 Marker:** bookmark for later recall
- **🌙 Rest:** sleep/relaxation trigger
- **🧠 Recall:** context reactivation
- **✨ Eureka:** new insight emerged

## Usage Pattern
1. Emoji status line (1–3 emojis max)
2. Structured bullets: Goal, Action, Result, Next
3. If AAK active: include “Reasoning Summary” and audit binding

## Examples
- "✅ ⚡" — OK, Quick path chosen → one step listed
- "⚠️ 🧪" — Caution, Test first → outline verification
- "🛑" — Stop Now → no output except “Stopped.”

## Cross‑Lingual Notes
- Provide bilingual mirrors when needed (en + fa + fa_finglish)
- Avoid ambiguous or culturally loaded emojis; default to above palette
